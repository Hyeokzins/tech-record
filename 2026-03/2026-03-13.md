## 폰트 판정 기능
### 정리
- 폰트 추적 기능 개발 과정에서 자동 매칭 시 "L"이 "F"로 매칭되는 현상이 발생함. 기술적 한계로 판단하고 OCR로 정답 텍스트를 먼저 인식한 뒤 1:1 매칭해야겠다고 생각했으나, 원인 분석 결과 PDF 좌표계(y 아래로 증가)와 TTF 좌표계(y 위로 증가)가 반대였던 것이 문제였음. y축 반전 전처리를 추가하자 정답 텍스트 없이도 정확한 매칭이 가능해졌고, 올바른 폰트 비교 시 유사도 0.9 이상이 나옴.
### 판정 케이스별 결과

- Verdana Bold → Verdana Bold 판정 (정답 텍스트 없음)
	- 글자가 뭔지 몰라도 형태만으로 가장 가까운 글리프를 자동 매칭. 글자별 매칭 정확도 높고 유사도도 높게 나옴.
- Verdana Bold → Verdana Regular 판정 (정답 텍스트 없음)
	-  폰트가 다르므로 글자별 매칭이 어긋남. 가안의 G 점 분포가 Regular의 G보다 Regular의 a에 더 가깝다고 판단하는 등 오매칭 발생. 전반적으로 유사도 낮음.
- Verdana Bold → Verdana Regular 판정 (정답 텍스트 있음)
	- 강제로 G↔G 1:1 비교를 해도 유사도가 낮게 나옴. 오매칭 원인도 카드 시각화로 직접 확인 가능.
### 폰트 검증에 필요한 것
- 검증하려는 폰트의 TTF 파일만 있으면 됨. 나머지는 가안 PDF에서 추출한 벡터 드로잉 데이터와 비교하면 끝. TTF 파일은 Windows에 기본 내장되어 있어 일반적인 폰트는 별도 확보 불필요. 특수 폰트도 해당 폰트를 실제로 사용하는 환경이라면 이미 TTF가 존재하므로 현실적으로 문제가 되지 않음.
### TTF 직접 비교 방식의 의의
- VLM이나 AI 모델을 거치지 않고 폰트가 직접 기록한 좌표 데이터끼리 수치 비교하는 방식이므로 정확도가 높음. VLM 방식은 속도가 느리고 정확도도 낮아 폰트 검증 용도로는 부적합함.
