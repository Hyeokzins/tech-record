## LG 라벨 도면 PDF 분석 탐색기 개발 
- 배경 및 목적
여러 국가의 라벨 도면 PDF를 자동으로 처리하는 시스템 개발 중 라벨 영역을 도면에서 정확히 찾아 크롭하는 기능이 필요해짐.
사람은 도면을 보고 테두리 사각형으로 라벨 영역을 직관적으로 인식하는데, 이를 자동화하려면 먼저 다양한 도면에서 라벨 테두리가 벡터 데이터 상 어떤 특징을 가지는지 파악해야 했음.

## 탐색 과정에서 발견한 패턴
- 여러 도면 PDF를 직접 클릭하며 분석한 결과:
라벨 테두리는 대체로 type: s (stroke only) 또는 type: fs (fill+stroke) 패스
단순 사각형(items: re 1개)이 일반적이나, 반원+직사각형 복합형(c+l 조합)도 존재
라벨 테두리는 도면 내 벡터 드로잉 중 면적이 가장 큰 것이라는 공통점 확인
OCG 레이어 정보는 대부분 None → 레이어 기반 필터링 불가


## 라벨 테두리 탐지 로직 변화
- 단계조건초기type:s + items==1 + re + 실선1차 완화type:s + 면적 최대 (복합형 대응)2차 확장type:s or fs (배경색 있는 라벨 대응)현재 테스트 중type 조건 제거, 면적 최대만
